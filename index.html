My Home Lab Journey — Building, Learning, and Experimenting

Practical notes on self-hosting, AI workflows and infrastructure as code.

Hi — I build things to learn how technology and AI workflows really work. These pages document experiments, real mistakes and the practical lessons that follow. This collection is a living record: short, practical, and reproducible.

Overview

My home lab is a compact, hands-on setup built from an old Dell XPS 9350 running Proxmox VE. It’s where I test virtualization, containers, networking and AI acceleration — not for production, but to learn how systems fit together and to experiment with building useful prototypes.

What I built

  • Proxmox VE as the hypervisor on an old laptop (USB-C Ethernet dongle used for network).
  • Ubuntu Server VM as the Docker host (named dockerhost), with Portainer to manage containers.
  • Immich deployed via Docker Compose; AI tasks are accelerated by an NVIDIA RTX 2070 SUPER GPU available on my LAN/gaming rig.
  • Experiments with services such as AdGuard and Nextcloud informed my networking and DNS approach.

Networking & domain

Current state

  • Most remote access is via Tailscale for secure, private connectivity while I experiment.
  • I purchased mulin.dev and added it to Cloudflare; DNS/nameservers will be finalized once the site and reverse proxy setup are ready.
  • Goal: provide simple HTTPS access for friends and family using Cloudflare Tunnels + automated certificates and a small reverse proxy on the server.

Storage & resilience

I use a shared mount (e.g. /mnt/storage) mounted into the VM for uploaded files. Next steps include standardizing backups: VM snapshots, configuration in Git, and encrypted off-site mirrors for sensitive data.

Automation & next steps

  • Rebuild the stack reproducibly with Terraform and Ansible so the entire environment can be redeployed from scratch.
  • Continue documenting everything in an Obsidian vault to turn notes into a site and reproducible scripts.
  • Refine DNS, reverse proxy, and HTTPS for mulin.dev.

Why this matters

Beyond the technical skills, this project is how I learn to incorporate AI tools and workflows into practical prototypes. The ability to quickly prototype — sometimes without deep prior coding experience — changed how I approach problems, especially the kinds of operational challenges I saw while managing HVAC operations. These experiments are purposeful: they’re a way to practice turning ideas into working prototypes.
